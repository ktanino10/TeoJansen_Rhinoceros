# Contributing to Theo Jansen Rhinoceros Project

Thank you for your interest in contributing to this educational project! This guide will help you get started.

## About This Project

This is an educational project designed to:
- Share knowledge about kinetic sculptures
- Help people learn CAD design and 3D printing
- Inspire children to explore engineering and art
- Respect and honor Theo Jansen's original work

## How You Can Contribute

### 1. Report Issues

If you find problems with:
- Design files that don't work correctly
- Documentation that's unclear or incorrect
- Broken links or missing information
- STL files that don't print well

Please open an issue describing:
- What you expected to happen
- What actually happened
- Steps to reproduce the problem
- Your environment (printer, software versions, etc.)

### 2. Improve Documentation

Documentation contributions are especially welcome:
- Fix typos or clarify instructions
- Add assembly steps with photos
- Create tutorials or guides
- Translate documentation to other languages
- Add educational resources

### 3. Share Your Builds

If you've built this project:
- Share photos of your build
- Document any modifications you made
- Report what worked well and what didn't
- Suggest improvements

### 4. Enhance Designs

If you have improvements to the design:
- Describe the problem you're solving
- Explain your solution
- Provide modified CAD files
- Include new STL files if applicable

## Guidelines

### Educational Focus

Remember this is an educational project:
- Keep instructions clear and beginner-friendly
- Explain the "why" not just the "how"
- Consider diverse skill levels
- Maintain the learning-focused tone

### Respect for Original Work

Always respect Theo Jansen's original Strandbeest designs:
- This project is inspired by, not copying, his work
- Maintain educational and non-commercial purpose
- Encourage others to learn about his original creations
- Give proper attribution

### Code of Conduct

Be respectful and constructive:
- Be kind and welcoming to all contributors
- Focus on ideas, not individuals
- Accept constructive criticism gracefully
- Help create a positive learning environment

## Submitting Changes

### For Small Changes

For minor fixes (typos, small documentation updates):
1. Fork the repository
2. Make your changes
3. Submit a pull request with a clear description

### For Larger Changes

For significant changes (new features, major redesigns):
1. Open an issue first to discuss your idea
2. Wait for feedback before investing significant time
3. Fork the repository
4. Make your changes in a feature branch
5. Submit a pull request with detailed description

### Pull Request Guidelines

When submitting a pull request:
- Provide a clear title and description
- Explain what changes you made and why
- Reference any related issues
- Include photos/screenshots if relevant
- Test your changes if applicable

## File Organization

Please maintain the repository structure:
```
Fusion360/          # CAD source files
STL/                # 3D printable files
docs/               # Documentation
  images/           # Photos and diagrams
  ASSEMBLY.md       # Assembly instructions
README.md           # Main documentation (English)
README_JP.md        # Japanese documentation
```

## Design File Formats

- **CAD Files**: Fusion 360 (.f3d) format preferred
- **3D Prints**: STL format for maximum compatibility
- **Images**: PNG or JPEG, reasonably sized
- **Documentation**: Markdown (.md) format

## Questions?

If you have questions about contributing:
- Open an issue with the "question" label
- Check existing issues for similar questions
- Be patient - this is maintained by volunteers
